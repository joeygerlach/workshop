Using a NeCTAR Cloud Virtual Machine

February 2017

This guide will show you how to use your NeCTAR VM to perform jobs, including tips on making efficient use of cloud capabilities.

This tutorial will demonstrate some basics of setting up and running a VM. We will install packages, download files, and run a demonstration analysis with some optimisation tips.

The aim of the workshop is to make using a NeCTAR cloud a less intimidating experience by walking through some of the common steps in running analyses on a VM.

Launch a Virtual Machine (Instance)

This tutorial will provide an overview of launching and accessing a virtual machine (VM) in NeCTAR. These processes are covered in depth on the NeCTAR website Training Modules http://training.nectar.org.au/ , and Support Pages https://support.ehelp.edu.au/support/solutions

Instructions in <arrows> should be replaced. (e.g. <path/to/key> may be replaced with  ~/.ssh/key.pem )

 

1.        Log on to the NeCTAR dashboard https://dashboard.rc.nectar.org.au/project

Select the correct project allocation in the top bar (if it starts with "pt- ", it is your default trial allocation)

 

2.        Select "Access & Security" under the "Compute" subheading in the left side main menu.

·       Navigate to “Security Groups” and click "Create Security Group"

·       Name the security group 'SSH', with the description "port 22 for SSH". Click "Create Security Group"

·       Click "Manage Rules" in the "Actions" drop-down menu. Click "Add Rule".

·       Type "22" under "Port". Click “Add”

 

Optional: Under "CIDR", you can choose a range of IP adresses that can access your VM through port 22. If you keep it as "0.0.0.0/0", you can access your VM from any computer, but others might find a way to access it also. Here are the IP ranges for SA University networks. Use one of these, and the VM can only be accessed form a University computer (including an external computer using a VPN to access the University network).

•          129.127.0.0/16 - University of Adelaide

•          129.96.0.0/16 - Flinders

•          130.220.0.0/16 – UniSA

 

3.        Select “Access and Security”, then the "Key Pairs" tab.

a)        Click “Create Key Pair”, name it and observe that a ___.pem file is automatically downloaded.

b)        Save the private key (.pem file)  to your computer in a place you can easily find again.

·       MacOS users: enter the following in your Terminal app : chmod 600 <path/to/key>

·       Windows users use Puttygen to convert key to ___.ppk file (see demo instructions)

See launching-virtual-machines for more instructions on setting up keypairs.

 

4.        Select "Instances" in the left side main menu. Click "Launch Instance".

a)        Give your VM a name, choose a "Flavor" (size of the VM), and select an Image (pre-loaded operating system, e.g. NeCTAR Ubuntu 16.04).

b)        Select the "Access & Security" tab. Select your Key Pair name, and selec

c)        t the "SSH" security group.

d)       Click "Launch". It may take a few minutes for the instance to boot.

 

Connect to your VM

For full instructions, see the support page accessing-instances

Copy the IP Address of the instance you have created (found in the “Instances” tab on the dashboard).

·       Windows - Set up a PuTTY connection with the IP address and the username “ubuntu”

·       Mac/Linux - In the Terminal app, enter ssh -i <Path/to/key> ubuntu@<IPaddress>

Some commands to run the first time you access your VM

sudo passwd ubuntu - Set a password for the user "ubuntu" (tip: the password is typed invisibly)

sudo chown ubuntu /mnt - Make user "ubuntu" the owner of the mounted storage in /mnt (Only if you have launched a flavor that has "Ephemeral Disk" storage). Without this command, you cannot save data to the larger storage disk.

•          Run the following commands periodically to update the package manager and the installed packages. We won't run them today because they can be time-consuming.

sudo apt-get update

sudo apt-get upgrade

To close the SSH session, enter the command exit

Launching a Remote Desktop Instance

By installing some programs on the VM and your PC, we can create a GUI virtual desktop to interact with the VM. We will launch an instance with an image that is already set-up with X2Go server and Mate pre-installed. If the X2Go client is already installed on your PC, you are ready to launch a remote destop VM.

X2Go website ; Installing the X2Go client *NB MacOSX users will need to install XQuartz first if they don't have X11 server installed.

 

Dashboard – Launching an instance form a pre-configured image

1.        On the Nectar Dashboard, select the Images tab, then select Public.

2.        Click on the column header Image Name so the column is alphabetised

3.        Find the image named TPAC core.0XX XXXXXXXXXX and select Launch Instance

4.        Give the instance a name and choose a suitable Flavor (m2.small suggested for the tutorial)

5.        Under Access and Security select the keypair and security group you set up earlier.

6.        Launch Instance

 

X2GO – Connect to the instance with a remote desktop

7.        Open the X2Go client. Click Session, then New Session.

8.        Choose a session name.

9.        Copy and paste the IP address of the instance for Host. Login is "ubuntu"

10.     Browse for the key you saved (.pem) for ssh connection.

11.     Under Session Type, select "MATE"

12.     Select the Shared folders tab at the top of the window.

13.     Browse for a folder on your PC that you can share with the VM. Select Add and click Automount.

·       MacOS – you might need to go to your computer’s “system preferences / sharing” and select file sharing

14.     Ensure "use SSH port forwarding..." is selected and click OK.

15.     Select the session in your X2Go client main window and it will connect.

 

Look around the remote desktop environment. Your shared folder will be in "ubuntu (home) / media / disk /". Right click and "make a link" that you can put in your Ubuntu (home) folder.

In the top left, click "Applications", then "System Tools" and select MATE Terminal. This is the command-line interface within the remote desktop session.

The following instructions will be used identically whether via SSH connection through your PC's terminal or PuTTY application, or through the Mate Terminal application in your remote desktop session.

However, some commands could be replaced with "point-and-click" processes in your remote desktop session. E.g. downloading program files using the FireFox web browser rather than the wget command; copying and pasting.

 

 

Running an example data analysis job

We will be running a job with the population genetic program "Structure". This program is a good example of an "embarrassingly parallel problem" that is ideal for running on a cloud VM with a large number of processors.

Downloading and decompressing files

•          To download "Structure", copy the URL for the latest release http://pritchardlab.stanford.edu/structure_software/release_versions/

•          Then use the "wget" command to download the tar.gz file. e.g.
wget http://pritchardlab.stanford.edu/structure_software/release_versions /v2.3.4/release/structure_linux_console.tar.gz

•          Decompress the archived folder with:
tar -zxvf structure_linux_console.tar.gz

•          Download today's tutorial file:
wget https://github.com/joeygerlach/structure_parallel/archive/master.zip

•          Use the unzip command to decompress the tutorial file
sudo apt-get install zip unzip
unzip master.zip

Moving the analysis to the mounted storage "/mnt"

Always perform analyses in a larger storage disc, not the primary (root) disc. In this tutorial we will use the secondary (ephemeral) disc, but some users may request volume storage to use for running analyses instead because the data will be more secure.

•          First, we make the mounted storage disc writable:
sudo chown ubuntu /mnt

•          Copy the tutorial files to the mounted disc:
cp -r ~/structure_parallel-master/ /mnt/structure_run1/
cd /mnt/structure_run1

•          Look at the files (you may wish to use FileZilla for this)
ls -lh

Running the program - using "Parallel" and "nohup"

 

•          Look at the support page on running jobs on a VM:
https://support.nectar.org.au/support/solutions/articles/6000089713-tips-for-running-jobs-on-your-vm

•          Install the task manager "GNU Parallel"
sudo apt-get install parallel

•          Run Structure using "GNU parallel" to split the jobs amongst the available CPUs:
parallel < example_commands.txt

•          Look at top, htop or the System Monitor to see the job running.

•          When you close the terminal (or PuTTY) window, the job will be terminated. In order to keep the process running after we have logged out, we add nohup to the command. nohup parallel < example_commands.txt 2>&1 &

 

Transferring files

We have created a folder of results files. We will transfer them to your computer, or to a remote data storage server.

First, we can zip the results folder into a single file.
zip structure_1.zip results OR
tar -zcvf structure_1.tar.gz results/

Filezilla

Drag and drop files between your VM and your local computer.

Secure File Transfer Protocol

Use the sftp command to access a remote computer, then use "get" to download or "put" to upload files.

As an example, if you have storage at eRSA you may use:

sftp <username>@sftp.ersa.edu.au
get test.txt ./
put test.tar.gz ./
quit

Terminating Instances

Remember to terminate instances when they are no longer being used. This can be done in the "Instances" tab of the Nectar dashboard. Remember to save any data to a secure data storage location first. You may choose to create a "snapshot" of the instance before terminating it, in case you need to revisit the compute environment for future analyses.

Quick reference for shell commands

Connecting from the terminal app on your Mac:

Commands for your Linux VM

Command

Action

sudo passwd ubuntu

set a password for user 'ubuntu'

sudo chown ubuntu /mnt

make the ephemeral disk writable

lsblk

list the block storage

df -hT

display the disk usage

du -h <path/to/directory>

display directory and file sizes

top or htop

activity monitor for your VM

ps

list the running processes on your VM (with PID#)

kill <PID#>

terminate the process by PID number

control + 'c'

stops a process running in your terminal

sudo apt-get update

updates the list of packages available to install

sudo apt-get upgrade

upgrades the installed packages

apt-cache search <name>

search for a package to install

sudo apt-get install <name>

install a package

parallel < commands.txt

distribute a list of commands over multiple CPUs

nohup <normal commands go here> 2>&1 &

keep a job running in the background
